### 三、实验内容

#### 3.1 必做实验——算术练习软件（ArithmeticPractice.java）

##### 3.1.1 功能概述

基于 Java Swing 开发了一款面向小学生的算术计算练习软件。程序提供友好的图形界面，支持自定义难度、随机出题、即时判对错和成绩统计功能。

##### 3.1.2 核心功能

**(1) 自定义难度**

程序提供三级难度选择：
- **简单模式**：操作数范围 1～20，适合低年级学生
- **中等模式**：操作数范围 1～100，适合中年级学生
- **困难模式**：操作数范围 1～500，适合高年级学生

不同难度不仅影响数值范围，还会影响：乘法运算中小数值的乘数范围、除法运算中被除数和除数的范围。

**(2) 随机出题**

程序调用 `java.util.Random` 生成随机操作数，并根据用户选中的运算类型（加、减、乘、除）随机选择一种运算。对特殊情况进行如下处理：
- **减法**：确保被减数 ≥ 减数，避免出现负数（小学生友好）
- **除法**：通过逆运算（先确定商和除数，再计算被除数 = 商 × 除数）确保结果为整数，避免出现余数
- **乘法**：简单和中等模式下限制乘数范围（最大12），降低计算难度

**(3) 数字类型支持**

- **整数模式**：操作数和答案均为整数
- **小数模式**：操作数随机生成一位小数（如 3.5、12.7），答案精确到小数点后两位

**(4) 判断与反馈**

- 用户在输入框中填写答案，点击「提交答案」按钮或按回车键提交
- 系统将用户答案与正确答案比较（小数模式允许 0.001 的精度误差）
- 正确：输入框边框变绿 + 显示 "🎉 正确！"
- 错误：输入框边框变红 + 显示正确答案
- 答题后输入框禁用，防止重复提交

**(5) 计分与统计**

底部状态栏实时显示：
- 📊 已答题数 / 正确题数
- 🎯 正确率（百分比），颜色区分：
  - ≥90%：绿色（优秀）
  - 60%～90%：橙色（良好）
  - <60%：红色（需加油）

##### 3.1.3 界面设计

采用 **CardLayout** 实现双页切换：

**设置页**：
- 软件标题 "🧮 算术练习小能手"
- 难度下拉选择框、数字类型下拉选择框
- 加减乘除四种运算的勾选框
- 「🚀 开始练习」按钮

**游戏页**：
- 顶部导航栏（标题 + ⚙设置按钮）
- 白色圆角题目卡片，大字体显示算式
- 答案输入框 + 提交按钮
- 结果提示标签（带颜色反馈）
- 下一题按钮
- 底部状态栏（题数 + 正确率）

**配色方案**：采用柔和的 Material Design 风格配色，主色靛蓝 (#5C6BC0)，强调色珊瑚橙 (#FF8A65)，成功绿 (#66BB6A)，错误红 (#EF5350)，背景浅灰 (#F5F7FA)。

**交互细节**：
- 按钮悬停时颜色变亮
- 输入框支持回车快捷提交
- 答完一题后焦点自动跳转到「下一题」按钮
- 窗口居中显示，设置最小尺寸 600×500

#### 3.2 选做实验——终端扫雷游戏（Minesweeper.java）

##### 3.2.1 功能概述

基于 Java 控制台开发了一款完整的扫雷游戏，支持三级难度、计时计步、保存/读取进度等功能。使用 Unicode 字符和 ANSI 转义序列实现了丰富的终端视觉效果。

##### 3.2.2 核心功能

**(1) 难度系统**

| 难度 | 棋盘大小 | 雷数 |
|------|----------|------|
| 🟢 初级 | 9 × 9 | 10 |
| 🟡 中级 | 16 × 16 | 40 |
| 🔴 高级 | 16 × 30 | 99 |

**(2) 游戏规则**

- 使用 `WASD` 键移动光标，光标位置以反色高亮显示
- `空格键` 翻开当前格
- `F` 键插旗标记地雷，再次按 `F` 取消旗子
- 翻开数字格显示周围 8 格的地雷数量（1-8）
- 翻开空格（周围无雷）自动递归扩散，翻开整片安全区域

**(3) 首次安全保证**

地雷在玩家第一次翻开操作**之后**才布置，且保证第一步所在格及其周围 3×3 区域内没有雷，确保玩家永远不会第一步就踩雷。

**(4) 数字颜色系统**

```
1=蓝色  2=绿色  3=红色  4=深蓝  5=深红  6=青色  7=黄色  8=紫色
```

不同数字使用不同 ANSI 颜色，方便玩家快速识别危险区域。

**(5) 计时与计步**

顶部信息栏实时显示：
- ⏱ 游戏已用时间（MM:SS 格式）
- 🐾 操作步数（翻格 + 插旗均计数）
- 🚩 已插旗数 / 总雷数

**(6) 存档系统**

使用 Java 对象序列化（`ObjectOutputStream` / `ObjectInputStream`）实现游戏进度保存和恢复：
- `P` 键保存当前游戏到 `saves/minesweeper_YYYYMMDD_HHmmss.sav`
- 同时保存一份 `saves/minesweeper_latest.sav` 作为快捷读取
- 主菜单选择「4. 💾 读取存档」，列出所有存档文件供选择
- 存档数据结构包含：棋盘状态、地雷布局、光标位置、步数、时间等完整信息

**(7) 终局判定**

- **胜利**：所有非地雷格均被翻开，显示 "🎉🎊 恭喜你赢了！🎊🎉"
- **失败**：翻开地雷格，显示 "💥 踩到地雷了！游戏结束！💥"，并揭示所有地雷位置，错误插旗处显示 ✖

### 四、心得体会

通过本次实验，我对 Java 面向对象程序设计有了更深入的理解和实践：

**（一）Swing GUI 编程方面**

1. **布局管理器的灵活运用**：CardLayout 实现了设置页和游戏页的无缝切换，比创建多个 JFrame 更加优雅；BoxLayout 和 BorderLayout 的组合使用使得界面布局清晰、响应式良好。

2. **事件驱动编程**：深刻理解了"事件源 → 监听器 → 事件处理"的三要素模型。按钮点击（ActionListener）、键盘输入（KeyAdapter）、鼠标悬停（MouseAdapter）等多种事件类型在同一个程序中协调工作，体会到了 GUI 程序「以事件为驱动」的编程范式。

3. **用户体验细节**：实现了输入框回车提交、答题后焦点自动跳转、正确/错误颜色反馈、悬停变亮效果等细节，让我认识到一个好的软件不仅要功能正确，更要在交互体验上下功夫。

**（二）扫雷游戏开发方面**

1. **算法思维**：递归展开空白区域（`reveal()` 方法的深度优先搜索）是扫雷的核心难点，实践了"当前格周围雷数为 0 时，自动递归翻开相邻 8 格"的算法逻辑。

2. **数据持久化**：通过 Java 序列化机制（`Serializable` 接口 + `ObjectOutputStream`）实现了游戏存档功能，这是第一次在实际项目中使用对象序列化，理解了 transient 关键字的含义和应用场景。

3. **终端 UI 设计**：利用 ANSI 转义序列实现了颜色输出（不同数字用不同颜色）、反色高亮光标、清屏等效果，极大提升了终端应用的用户体验。同时也认识到跨平台（Windows 终端对 ANSI 的支持差异）的兼容性问题。

**（三）综合收获**

本次实验两个项目总计约 700 行 Java 代码，涵盖了 Swing GUI、事件处理、多线程（SwingUtilities.invokeLater）、文件 I/O、序列化、递归算法、随机数生成等多个知识点，是一次全面的面向对象编程综合训练。通过从需求分析 → 功能设计 → 编码实现 → 测试调试的完整开发流程，提升了独立分析和解决问题的能力。
